## 1. Install

```bash
pip install debugerai
```

For the spaCy NER model (improves entity coverage signal):
```bash
python -m spacy download en_core_web_sm
```

## 2. Diagnose your first failure

```python
from debugai import analyze

result = analyze(
    prompt="What does Section 4 of the contract require?",
    output="Section 4 requires arbitration in Delaware under the Marbury Clause.",
    chunks=["Section 4 covers confidentiality.", "Governed by California law."],
    similarity_scores=[0.66, 0.59],
    temperature=0.75,
    explain_with_llm=False,
)

print(result["primary"]["failure"])     # hallucination
print(result["primary"]["confidence"])  # 0.95
print(result["primary"]["fix"])
# "Add grounding constraints: answer only from provided context;
#  cite sources; say 'not found' when unsupported."
```

## 3. Wrap your LLM client

Drop one line into your existing code:

```python
from openai import OpenAI
import debugai

client = debugai.wrap_llm(
    OpenAI(),
    on_diagnosis=lambda d: print("⚠️", d["primary"]["failure"]) if not d["healthy"] else None,
)

# Call your LLM exactly as before — nothing else changes
resp = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What is our refund policy?"}],
    # Attach RAG context for full signal coverage:
    debugai_chunks=retrieved_docs,

)
# Diagnosis runs in the background — zero added latency
```

## 4. Use the universal completion API

Works with any provider — same call, any model:

```python
import debugai

resp = debugai.completion("gemini-2.0-flash", messages=[...])
resp = debugai.completion("ollama/qwen2.5", messages=[...])
resp = debugai.completion("groq/llama-3.3-70b-versatile", messages=[...])

print(resp.text, resp.cost_usd, resp.latency_ms)
```
